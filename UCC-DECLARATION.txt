UCC-DECLARATION – Declaration Contract
=======================================

TIER
-----
Tier 0 – Primitive. No dependencies on other UCC specs.

SCOPE
-----
This document defines the declaration contract: the formal interface
by which a component specifies a desired target state.

------------------------------------------------------------

1. FORMAL DEFINITION
------------------------------------------------------------

A declaration specifies the desired target state.

D : State → State ∪ {⊥}

  State — the desired state to converge to
  ⊥     — the transition is structurally impossible and permanent

------------------------------------------------------------

2. DECLARATION SEMANTICS
------------------------------------------------------------

A declaration MUST describe a target state, not an action.
It answers "what should exist" not "what should happen".

A declaration is evaluated against the observed state S0 to produce
the desired state S1 = apply(declaration, S0).

If S1 = ⊥, the transition is impossible:
  - If the impossibility is permanent → outcome=failed, failure_class=permanent
  - If the impossibility is transient → outcome=failed, failure_class=retryable

------------------------------------------------------------

3. DECLARATIVITY REQUIREMENT
------------------------------------------------------------

A declaration MUST be:
  - Declarative: describes desired state, not procedure
  - Side-effect free: evaluation MUST NOT modify any system state
  - Deterministic: given the same S0, it always produces the same S1

------------------------------------------------------------

4. IDEMPOTENCE
------------------------------------------------------------

The fundamental guarantee of the declaration contract:

  apply(declaration, apply(declaration, S)) = apply(declaration, S)

This guarantees safe retries, crash recovery, and distributed
orchestration without application-level deduplication guards.

------------------------------------------------------------

5. NORMATIVE RULES
------------------------------------------------------------

DECL-1 – Declarativity
  A declaration MUST NOT describe an action, only a target state.

DECL-2 – No side effects
  Declaration evaluation MUST NOT perform side effects.
  It is a pure function of the observed state.

DECL-3 – Determinism
  Given identical observation and declaration, the result MUST be
  identical.

DECL-4 – Idempotence
  Applying the same declaration twice MUST produce the same state.

------------------------------------------------------------

6. MINIMAL INTERFACE
------------------------------------------------------------

apply(declaration, State) → State | ⊥
