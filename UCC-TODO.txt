UCC-TODO.txt
============

PURPOSE
-------
Actionable task list for closing the gaps identified in UCC-GAPS.txt.
Each task maps to one or more gap entries and describes exactly what
needs to be written, defined, or built.

STATUS
------
Active — updated as tasks are completed or new gaps are found

CLASSIFICATION
--------------
Reference – Framework Work Backlog

------------------------------------------------------------

HOW TO READ THIS DOCUMENT
--------------------------
Tasks are grouped by area and ordered by dependency (earlier tasks
unblock later ones). Each task has:

  ID      — stable identifier (TODO-N)
  GAP     — reference to UCC-GAPS.txt entry
  AREA    — what part of the framework it affects
  TASK    — exactly what needs to be done
  OUTPUTS — what documents/sections are created or updated
  BLOCKS  — other TODOs that depend on this one

Task states (update in place):
  [ ] open
  [~] in progress
  [x] done

[ ] TODO-H2
    CON:     Governance ceremony (model_binding, version blocks,
             record immutability) exceeds benefit for small systems.
    AREA:    UCC-IMPL-GUIDE.txt, UCC-CHECKLIST.txt
    TASK:    Clearly mark which compliance items are mandatory for
             ALL components vs. which are mandatory only at RIG grade
             or when using the full record model. A single-service
             UCC+Basic component should have a minimal compliance surface.
             Add a COMPLIANCE TIERS table to UCC-CHECKLIST.txt:
               Tier 1 (mandatory always):   result model, schema validation
               Tier 2 (mandatory at RIG):   state references, test contract
               Tier 3 (mandatory at scale): record model, model_binding
    OUTPUTS: UCC-CHECKLIST.txt — COMPLIANCE TIERS table
             UCC-IMPL-GUIDE.txt — note on minimal vs. full compliance

[ ] TODO-H3
    CON:     No ecosystem — no libraries, no validators, no IDE support,
             no community. Every team starts from scratch.
    AREA:    Product / ecosystem
    TASK:    Define the minimal ecosystem needed to make adoption
             self-sustaining:
             1. A published JSON Schema file (see TODO-E1)
             2. A language-agnostic reference implementation (see TODO-F1)
             3. At least one library wrapper in a common language
                (Python or TypeScript) exposing observe/diff/apply
             4. A README or getting-started page that links
                UCC-GUIDE-MINIMAL.txt as the entry point
